简答题

  1. 简述瀑布模型，增量模型，螺旋模型（含原型方法），并分析优缺点

      1. 瀑布模型
        优点：
        定义了软件开发基本流程与活动，包括创意阶段，描述问题，市场，关键技术等
        缺点：
        1. 依赖问题，后面的工作依赖于前一阶段的工作
        2. 容错问题，后期开发测试发现需求有问题需要推倒重来，这个工作量难以接受
        3. 资源调配问题，不同团队，不同项目的知识技能需求不同，人员数量要求不同
        总得来说，瀑布模型容易导致项目延期，项目不可控
      2. 增量模型
        优点：
        1. 人员配置灵活
        2. 降低系统失败的风险
        3. 可以给客户一个包含部分功能的版本，增强客户对该项目的信心
        缺点：
        1. 难以确定所有的基本业务服务
        2. 并行开发构件存在不能集成的可能性
      3. 螺旋模型（含原型方法）
        优点：
        1. 客户可以跟进项目开发，一方面有助于客户了解项目的进度，另一方面有助于
         开发人员和客户的沟通
        2. 设计灵活，可以在项目的各个阶段进行变更
        3. 成本计算变得更加容易
        缺点：
        1. 过多的迭代可能会拖延发布的时间
        2. 不适合应用于小规模的软件开发
        3. 错误的风险分析会带来更大的风险，所以要求相当丰富的风险评估知识

  2. 简述统一过程的三大特点，与面向对象的方法有什么关系

      1. 以架构为中心
      2. 迭代式增量开发
      3. 用例驱动
      与面向对象的关系：
       统一过程面向功能设计接口，基于设计的类进行编程，这是面向对象编程的特征

  3. 简述同意过程4个阶段的划分准则是什么？每个阶段的关键的里程碑是是什么？

      1. 初始阶段：大体的猜想，业务案例，范围以及模糊的评估
        里程碑：生命周期目标里程碑
      2. 细化阶段：分析问题，确定大多数的需求和范围，进行更为实际的评估
        里程碑：生命周期体系里程碑
      3. 构建阶段：将剩余的构建和功能开发并整合为一个软件产品
        里程碑：初始运行能力里程碑
      4. 交付阶段：测试部署以及交付
        产品发布里程碑
      每一个阶段结束于一个里程碑，在结尾进行一次评估，判断是否合格可以进入下一个
      阶段

  4. 软件企业为什么能按固定节奏生产，固定周期发布软件产品？它给企业项目管理带来哪些好处？

      通过统一过程，在开发产品之前，企业可以将开发的过程划分为若干个阶段。在每一
      个阶段的结尾进行评估，以判断是否合格可以进入下一个阶段。通过这样将过程划分
      为阶段，阶段之间设置里程碑，企业可以准确掌握开发的时间以及完成的时间，从而
      可以按照固定的节奏进行生产，发布产品。
      企业可以按照固定节奏生产，有利于企业保持活力，跟上市场的变化，变得更加有竞
      争力。
